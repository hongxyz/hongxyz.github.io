在linux下安装anaconda 并使用anaconda安装pytorch

发表于 2018-11-15

在linux上安装anaconda 使用anaconda安装pytorch

安装anaconda

由于我使用的是实验室的服务器，所以使用anaconda来隔离各自的环境，而且我也咩有root权限，搞不来pip2333
首先是在anaconda官网下载linux的版本 2.7和3.7都可以，反正后面如果一些包版本不支持3.7的话也会给你下一个3.6或者其他依赖的python版本
bash Anaconda3-5.3.0-Linux-x86_64.sh
安装anaconda，首先是过一遍license，接着安装，安装完之后会问你需不需要把anaconda的环境设置搞定，写在.bashrc里面，要选是，如果不小心跳过了，可以选择no后里面的提示，去设置，我记得是一句source什么什么的，忘了
后面的visual code选择不装
接着！！！最重要一点是要 source ~/.bashrc ，去激活anaconda的环境设置，不然就用不了，当然，关了终端重新开就也可以了
source之后，用conda list查看已安装的包，再输入python看看python是不是使用的anaconda的，而不是使用linux给的python
如果conda list时候告诉你没有conda命令，记得看一下是否source ~/.bashrc了

另外一些坑点

输入python后显示
no module named _sysconfigdata_nd
网上看了主要是python本身的bug，要解决需要root权限，我又没有…
最终解决方法是重新安装anaconda，并且发现
如果你下载的anaconda的python版本和linux里面的版本一样，那么在调用python的时候就可能出现这种情况，选择一个不同的版本下载，让他自己给你设置路径就好了

安装pytorch

首先创建一个anaconda环境，每一个环境有不同的包这样相互隔离不会串来串去
conda create -n 环境名 python=版本号
比如
conda create -n pytorch python=3.6
激活
conda activate pytorch或者source activate pytorch
想要退出也很简单 conda deactivate 即可 或者source deactivate
在pytorch官网选择对应版本，我的是cuda9.2，python3.7，故而选择
conda install pytorch cuda92 -c pytorch
pip3 install torchvision

等待安装，不过因为anaconda下载东西是在anaconda.org的服务器上下载，所以非常慢
可以选择更改代理为国内镜像网站 ，比方说清华，中科大的等
conda config –add channels https://mirrors.ustc.edu.cn/anaconda/pkgs/free/
conda config –set show_channel_urls yes
更改镜像记得去 .condarc 删除原有的镜像

安装完后，python一下，试试import torch import torchvision怎么样，没报错就没有问题

坑点：

可能报错：GLIBC_2.14’ not found或者是GLIBC更高版本的错误
解决方法：这主要是自己机器的glibc的版本太低了，如果想要修改机器的glibc搞不太好的话有可能机器整个dump掉了
而且如果是在实验室服务器上，也没有root权限，所以可以采用以下解决方法
从 http://ftp.gnu.org/gnu/glibc/ 下载自己的版本，我自己下载了2.14.1版本
解压，并在文件夹创建build文件夹
tar xvf glibc-2.14.1.tar.gz
cd glibc-2.14.1
mkdir build
cd build

把安装信息搞到build里面
../configure –prefix=/你想要把包安装到的位置/glibc-2.14.1 –with-headers=/usr/include
网上有一些人是
../configure –prefix=/usr/local/glibc-2.14 –with-headers=/usr/include
../configure –prefix=/opt/glibc-2.14 –with-headers=/usr/include
但是你如果没有这些文件夹的权限的话，建议自己新建一个有权限的文件夹来放
比方说我自己是
../configure –prefix=/home/hong/glibc_temp/glibc-2.14 –with-headers=/usr/include
相当于指定把glibc安装在home/hong/glibc_temp/glibc-2.14里面

安装
make
make install
如果没有usr/local opt的权限的话，使用上面的两条就会报permission denied的错误

安装完毕之后，为了防止污染其他环境，输入命令即可，不要去改文件，反正，我没有root权限也改不了2333
命令
export LD_LIBRARY_PATH=/你安装的包的位置/lib:$LD_LIBRARY_PATH

之后再import torch ，问题解决

另外的坑点

如果你发现transform.Resize什么的报错说乜有这个attribute，就把transforms整个文件复制到你的目录再去import transforms就行了

two stream 论文笔记

发表于 2018-10-17

two stream 论文笔记

论文地址 https://arxiv.org/abs/1406.2199
名称 Two-Stream Convolutional Networks for Action Recognition in Videos
之前确定导师之后就跑去浪了…现在稍微静下心来读读老师叫师兄给的一些关于视频行为识别最基础的论文…
Two-Stream Convolutional Networks for Action Recognition in Videos这篇论文发在2014的NIPS上面，实际上想法很简单，就是分成两个网络，一个使用光流作为输入，一个使用单帧作为输入，输出结果都是各个类的softmax概率，接着对两个网络的结果使用一个简单的融合得到最终结果
所以这个方法需要分别训练两个网络，不是one-stage的

论文主要内容

abstract：

视频行为识别主要挑战在于要capture到视频的物体(appearance)信息和运动信息，
文章三个贡献在于：
1 提出了two stream模型，将时间，空间网络结合起来
2 在稠密光流上训练得到的网络效果可以
3 结合多个数据集的多任务学习提高了准确率

introduction：

two stream的方法将模型分为两个网络，一个是空间网络，在某一个帧在去预测视频类别，因为时静止的单帧，所以主要处理的是物体信息
一个是时间网络，通过光流的方法在连续多帧中得到物体运动信息进行预测，主要是对于物体运动信息进行处理

relative work:

一类方法是手动设计的特征+分类器如SVM等，设计的特征需要capture到时空的特征
一类使用深度网络，使用连续多帧作为输入，或者是使用无监督的方法得到时空特征

two stream architecture

视频可以分为时间的部分和空间的部分
空间的部分就是一个一个单独的帧，里面包含了物体，场景的信息
时间的部分是视频多帧体现的运动信息，包含了物体，场景的运动信息

  • 空间网络
    没什么好说的，就是一个图像分类网络，什么样的行为跟物体的信息还是息息相关的,所以需要这个图像分类网络，而且本身他就可以到达一个不错的结果

  • 光流网络
    光流刻画了连续两帧之间的差别，表示了连续两帧之间同一个像素的运动情况，因此可以用来刻画连续帧的运动情况
    输入可以选择多种
    1 光流的stacking
    具体的方法就是先计算出L个连续帧的稠密光流，假设图像宽高为w,h
    这样得到了L个光流向量场，每一个向量都包含有两个分量，一个是垂直分量，一个是水平分量，所以规模是L×w×h×2,我们将光流向量场拆成垂直的部分和水平的部分，作为输入的channel，这样输入的channel就是2L个，规模是2L×w×h
    2 基于轨迹的stacking
    3 双方向的光流
    选择一个帧，往前反向做光流，往后正向做光流
    4 mean flow subtraction
    光流的值减去均值，这是因为光流的运动信息包含了不仅仅是物体运动信息，还有摄像机的信息，这样可以抵消一部分摄像机的运动带来的影响

多任务学习

主要是为了利用好两个数据集，一个是UCF101，一个是HMDB-51，但是两个数据集的类别一个是101类，一个是51类，所以需要将这两个数据集通过某种方式结合起来。论文在最后的全连接层上增加了两个独立的softmax分类层，一个用于UCF一个用于HMDB

实现细节

  • 卷积网络参数
    使用relu做激活函数，max-pooling使用3×3的滑动窗口，步长为2，使用了与alexnet相同的local response normalization
  • 训练
    使用mini batch的SGD，batch size是256，从256个训练视频中随机选取一个帧
    对于空间网络，直接使用这个帧，并进行随机的剪裁到224×224，进行数据增强的操作（RGB抖动，拉伸等等）
    对于时间网络，输入为224×224×2L，初始化学习率为1e-2,50K后为1e-3，70K后为1e-4，80K后停止训练
  • 测试
    测试的时候在视频中随机选取25个帧，每一个帧对四个角和中心进行cropping和filpping，得到10个输入，这样每一个视频有250个输入，随后使用他们的softmax输出取均值即可
    在ILSVRC-2012上预训练
  • 多GPU训练
  • 光流
    光流的获取是time-consuming的，最好是先将所有视频的光流都计算一遍，存储为jpeg格式，这样训练时间网络会快得多

Evaluation

这部分讲的主要是对于实验的一些设置的结果和准确率的结果
主要有以下结论：

  1. 空间网络在pre train model 上fine tune会比较好
  2. 光流stacking选取10帧效果最好，也就是L=10
  3. 使用多任务学习提高了准确率
  4. 两个stream网络的融合使用SVM比较好，但是平均值更经济实惠

结论和提高方向

没什么…

个人总结

至少读了这篇后对于action recognition有了一些了解，这个内容的挑战主要在于要得到物体的信息，和运动的信息并加以融合
可以看到这篇文章主要的贡献就是提供了使用光流作为输入的方法，实际上是使用光流对视频的时序信息进行建模，感觉上还是有种手动设计特征的感觉。深度神经网络在其中的作用仍然是一头进一头出的香肠加工厂，毕竟这是2014年的论文。
另外一个收获就是至少知道了光流是什么，为什么可以表达运动信息，同时文章对于算法的实现还有一些其他的东西可以学习，比方说使用多任务学习结合两个数据集的方法等等
